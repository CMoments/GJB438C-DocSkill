### `main`

主处理入口函数。

```python
def main(
    template_path: str,
    config_path: str,
    project_path: str,
    output_path: str
) -> None
```

**参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `template_path` | `str` | Word 模板文件路径 |
| `config_path` | `str` | config.json 文件路径 |
| `project_path` | `str` | project.json 文件路径 |
| `output_path` | `str` | 输出文件路径 |

**说明**

执行完整的文档生成流程：
1. 加载 `config.json` 和 `project.json`
2. 调用 `fill_document` 生成输出文档

**返回值**

无返回值，文档直接保存到文件系统。

**示例**

```python
from main import main

main(
    template_path='[08]软件需求规格说明-438C.docx',
    config_path='config.json',
    project_path='project.json',
    output_path='output/[08]软件需求规格说明-438C-filled.docx'
)
```

## 5. find_and_rep 模块

### `find_and_replace_in_paragraph`

在段落中查找并替换文本，保持格式。

```python
def find_and_replace_in_paragraph(
    paragraph: Paragraph,
    old_text: str,
    new_text: str
) -> bool
```

**参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `paragraph` | `Paragraph` | Word 段落对象 |
| `old_text` | `str` | 要查找的文本 |
| `new_text` | `str` | 替换后的文本 |

**返回值**

| 类型 | 说明 |
|------|------|
| `bool` | 是否成功替换 |

**说明**

处理两种情况：
1. **单 run 替换**：文本在单个 run 中，直接替换保持格式
2. **跨 run 替换**：文本跨越多个 run，清空所有 run 后在第一个 run 放入新文本

**示例**

```python
from docx import Document
from scripts.find_and_rep import find_and_replace_in_paragraph

doc = Document('template.docx')
for para in doc.paragraphs:
    find_and_replace_in_paragraph(para, "旧文本", "新文本")
doc.save('output.docx')
```

## 6. dynamic_process 模块

### `parse_dynamic_content`

解析动态内容，提取各个子项。

```python
def parse_dynamic_content(content: str) -> List[Dict[str, any]]
```

**参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `content` | `str` | 动态内容字符串 |

**返回值**

| 类型 | 说明 |
|------|------|
| `List[Dict[str, any]]` | 子项列表，每项包含 `num`, `name`, `content` |

**支持格式**

1. **接口格式**：`接口1: xxx / 接口2: xxx`
2. **数字编号格式**：`1. xxx / 2. xxx`

**示例**

```python
from scripts.dynamic_process import parse_dynamic_content

content = """
接口1：用户接口
- 输入：用户名、密码
- 输出：认证结果

接口2：数据接口
- 输入：查询条件
- 输出：数据列表
"""

items = parse_dynamic_content(content)
# 返回：
# [
#   {'num': 1, 'name': '用户接口', 'content': '...'},
#   {'num': 2, 'name': '数据接口', 'content': '...'}
# ]
```

### `insert_dynamic_sections`

插入动态章节。

```python
def insert_dynamic_sections(
    doc: Document,
    section_id: str,
    placeholder_data: Dict
) -> None
```

**参数**

| 参数 | 类型 | 说明 |
|------|------|------|
| `doc` | `Document` | Word 文档对象 |
| `section_id` | `str` | 章节标识（如 "3.2."） |
| `placeholder_data` | `Dict` | 占位符数据 |

**说明**

1. 查找占位符段落（包含 `{section_id}X` 的段落）
2. 解析内容为子项列表
3. 将占位符替换为第一个子项
4. 保留原有段落格式

**示例**

```python
from scripts.dynamic_process import insert_dynamic_sections

placeholder_data = {
    'content': '接口1：xxx\n接口2：yyy'
}
insert_dynamic_sections(doc, "3.3.", placeholder_data)
```

## 9. 命令行接口

### 基本用法

```bash
python main.py \
  --template "模板文件.docx" \
  --config "config.json" \
  --project "project.json" \
  --output "输出文件.docx"
```

### 参数说明

| 参数 | 缩写 | 说明 | 必需 |
|------|------|------|------|
| `--template` | `-t` | Word 模板文件路径 | 是 |
| `--config` | `-c` | config.json 文件路径 | 是 |
| `--project` | `-p` | project.json 文件路径 | 是 |
| `--output` | `-o` | 输出文件路径 | 是 |
| `--help` | `-h` | 显示帮助信息 | 否 |

### 示例

```bash
# 基本用法
python main.py \
  -t "[08]软件需求规格说明-438C.docx" \
  -c "config.json" \
  -p "project.json" \
  -o "output/SRS-filled.docx"

# 使用绝对路径
python main.py \
  --template "/path/to/template.docx" \
  --config "/path/to/config.json" \
  --project "/path/to/project.json" \
  --output "/path/to/output.docx"
```
