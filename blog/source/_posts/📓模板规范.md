# 模板规范说明

> JSON 模板结构定义与字段规范

## 1. 概述

本文档定义 word-fillter-438C 项目中使用的 JSON 模板结构和字段规范，包括 `config_template.json` 和 `project_template.json`。

## 2. config_template.json 规范

### 2.1 整体结构

```json
{
  "project": { ... },
  "document": { ... }
}
```

### 2.2 顶级字段

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `project` | Object | 是 | 项目详细信息 |
| `document` | Object | 是 | 文档封面信息 |

### 2.3 project 字段

项目详细信息定义，用于生成项目背景和上下文。

```json
{
  "project": {
    "overview": {
      "description": "项目概述（示例：本项目用于……）",
      "type": "text_generation",
      "content": "string"
    },
    "scope": {
      "description": "项目范围（示例：包含A/B/C模块）",
      "type": "text_generation",
      "content": "string"
    },
    "background": {
      "description": "项目背景（示例：由于业务发展需要，原有系统无法满足……）",
      "type": "text_generation",
      "content": "string"
    },
    "objectives": {
      "description": "项目目标（示例：提升XX效率、降低XX成本、实现XX自动化）",
      "type": "text_generation",
      "content": "string"
    },
    "features": {
      "description": "功能特性（示例：功能特性1、功能特性2、功能特性3）",
      "type": "text_generation",
      "content": "string"
    },
    "target_users": {
      "description": "目标用户群体（示例：企业内部管理人员、终端消费者、系统管理员）",
      "type": "text_generation",
      "content": "string"
    },
    "constraints": {
      "description": "项目约束（示例：需在Q2上线、预算上限XX万、必须兼容现有系统）",
      "type": "text_generation",
      "content": "string"
    },
    "assumptions": {
      "description": "假设与依赖（示例：假设服务器资源充足、依赖第三方API稳定）",
      "type": "text_generation",
      "content": "string"
    },
    "references": {
      "description": "参考资料（示例：《XX行业规范》、GB/T 8567-2006）",
      "type": "text_generation",
      "content": "string"
    },
    "terminology": {
      "description": "术语定义（示例：API-应用程序接口、SLA-服务等级协议）",
      "type": "text_generation",
      "content": "string"
    }
  }
}
```

#### project 字段定义

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `overview` | FieldDef | 是 | 项目概述 |
| `scope` | FieldDef | 是 | 项目范围 |
| `background` | FieldDef | 是 | 项目背景 |
| `objectives` | FieldDef | 是 | 项目目标 |
| `features` | FieldDef | 是 | 功能特性 |
| `target_users` | FieldDef | 是 | 目标用户群体 |
| `constraints` | FieldDef | 是 | 项目约束 |
| `assumptions` | FieldDef | 是 | 假设与依赖 |
| `references` | FieldDef | 是 | 参考资料 |
| `terminology` | FieldDef | 是 | 术语定义 |

### 2.4 document 字段

文档封面信息定义，用于替换 Word 模板封面页的占位文本。

```json
{
  "document": {
    "密级": {
      "replacement_pattern": "内部",
      "type": "original",
      "description": "位于文档首页",
      "content": "string",
      "default_content": "内部,公开,机密"
    },
    "阶段": {
      "replacement_pattern": "阶段：",
      "type": "original",
      "description": "位于文档首页",
      "content": "string",
      "default_content": "需求分析阶段、设计阶段、开发阶段、测试阶段、部署阶段"
    },
    "版次": {
      "replacement_pattern": "A 版",
      "type": "original",
      "description": "位于文档首页",
      "content": "string"
    },
    "外部型号+产品名称": {
      "replacement_pattern": "外部型号+产品名称",
      "type": "original",
      "description": "位于文档首页",
      "content": "string"
    },
    "产品型号": {
      "replacement_pattern": "产品型号-XXXX",
      "type": "original",
      "description": "位于文档首页",
      "content": "string"
    },
    "公司名称": {
      "replacement_pattern": "XXXXXXXX公司",
      "type": "original",
      "description": "位于文档首页",
      "content": "string"
    },
    "日期": {
      "replacement_pattern": "XXXX年XX月XX日",
      "type": "original",
      "description": "位于文档首页",
      "content": "string"
    }
  }
}
```

#### document 字段定义

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `密级` | FieldDef | 是 | 文档密级（内部/公开/机密） |
| `阶段` | FieldDef | 是 | 开发阶段 |
| `版次` | FieldDef | 是 | 文档版次 |
| `外部型号+产品名称` | FieldDef | 是 | 产品完整名称 |
| `产品型号` | FieldDef | 是 | 产品型号标识 |
| `公司名称` | FieldDef | 是 | 开发单位名称 |
| `日期` | FieldDef | 是 | 文档日期 |

### 2.5 FieldDef 字段定义

通用字段定义结构。

```typescript
interface FieldDef {
  description?: string;           // 可选：字段描述
  replacement_pattern: string;    // 替换模式（Word模板中的占位文本）
  type: FieldType;                // 字段类型
  content: string;                // 实际内容（填写时替换 "string"）
  default_content?: string;       // 可选的默认内容选项
}
```

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `description` | string | 否 | 字段描述说明 |
| `replacement_pattern` | string | 是 | Word 模板中待替换的占位文本 |
| `type` | string | 是 | 字段类型（见下表） |
| `content` | string | 是 | 实际填写内容，初始值为 `"string"` |
| `default_content` | string | 否 | 默认值选项，供用户参考 |

### 2.6 type 字段说明

| 值 | 说明 | 使用场景 |
|------|------|----------|
| `original` | 直接填入用户输入 | 简单文本字段（如密级、阶段） |
| `text_generation` | 根据描述生成文本 | 需要基于用户描述生成的字段 |

## 3. project_template.json 规范

### 3.1 整体结构

```json
{
  "template_info": { ... },
  "structure": { ... }
}
```

### 3.2 顶级字段

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `template_info` | Object | 是 | 模板元信息 |
| `structure` | Object | 是 | 文档章节结构 |

### 3.3 template_info 字段

模板元信息定义。

```json
{
  "template_info": {
    "name": "GJB 438C 软件需求规格说明书",
    "file": "[08]软件需求规格说明-438C.doc",
    "encoding": "gbk",
    "sections": []
  }
}
```

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `name` | string | 是 | 模板名称 |
| `file` | string | 是 | 模板文件名 |
| `encoding` | string | 是 | 文件编码 |
| `sections` | Array | 否 | 章节列表（预留） |

### 3.4 structure 字段

文档章节结构定义，基于 GJB 438C 标准。

```json
{
  "structure": {
    "1": {
      "title": "范围",
      "placeholders": [ ... ]
    },
    "2": {
      "title": "引用文档",
      "type": "table",
      "description": "...",
      "replacement_pattern": "..."
    },
    "3": {
      "title": "需求",
      "placeholders": [ ... ]
    }
  }
}
```

### 3.5 Section 定义

```typescript
interface Section {
  title: string;                        // 章节标题
  type?: "table" | "content";           // 可选：内容类型
  description?: string;                 // 可选：章节描述
  replacement_pattern?: string;         // 可选：替换模式
  content?: string;                     // 可选：章节内容
  placeholders?: Placeholder[];         // 可选：占位符列表
}
```

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `title` | string | 是 | 章节标题 |
| `type` | string | 否 | 内容类型：`table` 或 `content` |
| `description` | string | 否 | 章节描述 |
| `replacement_pattern` | string | 否 | 替换模式 |
| `content` | string | 否 | 章节内容 |
| `placeholders` | Array | 否 | 占位符列表 |

### 3.6 Placeholder 定义

占位符定义结构。

```typescript
interface Placeholder {
  id: string;                           // 章节编号
  name: string;                         // 章节名称
  type: PlaceholderType;                // 填写类型
  description?: string;                 // 可选：描述
  replacement_pattern: string;          // 替换模式
  content: string;                      // 内容
  generation_prompt?: string;           // 可选：生成提示
  default_content?: string;             // 可选：默认内容
  subsections?: { [key: string]: Subsection }; // 可选：子章节
}
```

| 字段 | 类型 | 必需 | 说明 |
|------|------|------|------|
| `id` | string | 是 | 章节编号（如 "1.1", "3.2.1", "3.2.X"） |
| `name` | string | 是 | 章节名称 |
| `type` | string | 是 | 填写类型 |
| `description` | string | 否 | 字段描述 |
| `replacement_pattern` | string | 是 | 替换模式 |
| `content` | string | 是 | 实际内容，初始值 `"string"` |
| `generation_prompt` | string | 条件 | 生成提示（type=content_generation 时推荐） |
| `default_content` | string | 否 | 默认内容选项 |
| `subsections` | Object | 否 | 动态子章节 |

### 3.7 type 字段说明

| 值 | 说明 | 使用场景 |
|------|------|----------|
| `original` | 直接填入用户输入 | 简单文本字段 |
| `text_generation` | 根据描述生成文本 | 需要基于描述生成的内容 |
| `content_generation` | 根据提示生成结构化内容 | 需要结构化生成的复杂内容 |

### 3.8 动态章节（Dynamic Sections）

动态章节用于数量可变的内容，如 3.2.X（软件能力需求）和 3.3.X（软件外部接口需求）。

#### 动态章节结构

```json
{
  "id": "3.2.X",
  "name": "软件能力需求",
  "type": "content_generation",
  "description": "3.2.X（软件能力）本条应标识必需的每一软件能力...",
  "generation_prompt": "请为软件设计3-8个核心功能模块，每个模块包括：功能名称、功能描述、输入、输出、性能要求、约束条件",
  "replacement_pattern": "本条应标识必需的每一软件能力...",
  "content": "string",
  "subsections": {
    "3.2.1": {
      "name": "",
      "type": "content_generation",
      "content": "string"
    }
  }
}
```

#### subsections 结构

```typescript
interface Subsections {
  [sectionId: string]: {
    name: string;
    type: "original" | "text_generation" | "content_generation";
    content: string;
  };
}
```

### 3.9 嵌套子章节

某些章节包含固定的子章节，如 3.10（计算机资源需求）。

```json
{
  "id": "3.10",
  "name": "计算机资源需求",
  "type": "content_generation",
  "description": "硬件和软件资源需求",
  "subsections": {
    "3.10.1": {
      "name": "计算机硬件需求",
      "type": "content_generation",
      "replacement_pattern": "本条应描述针对本软件必须使用的计算机硬件的需求...",
      "content": "string"
    },
    "3.10.2": {
      "name": "计算机硬件资源使用需求",
      "type": "content_generation",
      "replacement_pattern": "本条应描述本软件的计算机硬件资源使用需求...",
      "content": "string"
    },
    "3.10.3": {
      "name": "计算机软件需求",
      "type": "content_generation",
      "replacement_pattern": "本条应描述本软件必须使用或必须被并入本软件的计算机软件的需求...",
      "content": "string"
    },
    "3.10.4": {
      "name": "计算机通信需求",
      "type": "content_generation",
      "replacement_pattern": "本条应描述本软件必须使用的计算机通信方面的需求...",
      "content": "string"
    }
  }
}
```

## 5. 模板验证规则

### 5.1 config.json 验证

| 验证项 | 规则 | 错误级别 |
|--------|------|----------|
| 顶级字段 | 必须包含 project, document | Error |
| 字段完整性 | 每个字段必须包含 type, replacement_pattern, content | Error |
| content 值 | 不能为 `"string"` 占位符 | Error |
| type 值 | 必须是 `original` 或 `text_generation` | Error |
| replacement_pattern | 不能为空 | Warning |

### 5.2 project.json 验证

| 验证项 | 规则 | 错误级别 |
|--------|------|----------|
| template_info | 必须包含 name, file, encoding | Error |
| structure | 必须存在且非空 | Error |
| placeholder.id | 必须存在且唯一 | Error |
| placeholder.type | 必须是有效的类型值 | Error |
| placeholder.content | 不能为 `"string"` 占位符 | Error |
| subsections | 递归验证所有子章节 | Error |

## 6. 扩展指南

### 6.1 添加新字段

1. 在模板中定义字段结构
2. 设置 `replacement_pattern`（需与 Word 模板一致）
3. 设置适当的 `type` 值
4. 初始化 `content` 为 `"string"`
5. 可选添加 `default_content`

### 6.2 添加新章节

1. 在 `structure` 中添加新章节编号
2. 定义章节 `title` 和 `placeholders`
3. 为每个占位符设置完整属性
4. 更新验证脚本（如需要）

### 6.3 添加动态章节

1. 使用 `X` 作为可变编号（如 `3.2.X`）
2. 定义 `generation_prompt` 生成子项
3. 初始化 `subsections` 模板
4. 实现对应的解析和插入逻辑

### 6.4 添加嵌套子章节

1. 在 placeholder 中添加 `subsections` 字段
2. 定义固定编号的子章节
3. 为每个子章节设置完整的属性

## 7. 命名约定

### 7.1 字段命名

- 使用 **snake_case**（下划线分隔）
- 例：`replacement_pattern`, `generation_prompt`

### 7.2 ID 命名

- 使用 **点分层次** 编号
- 例：`1.1`, `3.2.1`, `3.10.4`

### 7.3 动态 ID

- 使用 **X** 表示可变部分
- 例：`3.2.X`, `3.3.X`
