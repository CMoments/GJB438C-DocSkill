# 系统架构设计文档

word-fillter-438C 架构设计说明

## 1. 概述
### 1.1 文档目的
本文档描述 word-fillter-438C 的系统架构设计，包括组件划分、数据流、模块交互关系等,为开发者提供系统级别的技术参考。
### 1.2 系统定位
word-fillter-438C 是一个基于 Claude Code 的技能(Skill),用于自动化生成符合 GJB 438C 标准的软件需求规格说明书。系统采用模板驱动、交互式问答的设计理念。
### 1.3 设计原则
| 原则 | 描述 |
|------|------|
| **模板驱动** | 基于 JSON 模板定义文档结构,支持灵活扩展 |
| **交互式填写** | 通过 AskUserQuestion 工具引导用户完成信息收集 |
| **格式保持** | 文档处理时保留原始 Word 格式 |
| **分层架构** | 配置层、处理层、文档层分离,便于维护 |
| **可扩展性** | 支持自定义模板和字段扩展 |

### 2.2 层次说明
#### 2.2.1 Skill Layer(技能层)
技能层是系统的入口点,由 `SKILL.md` 定义。负责:
- 定义 Skill 元数据(名称、描述、许可证)
- 描述工作流程和执行步骤
- 提供用户交互指引
- 定义字段类型和处理规则
#### 2.2.2 Configuration Layer(配置层)
配置层管理两类核心数据:
| 文件 | 用途 | 结构 |
|------|------|------|
| `config.json` | 项目信息和文档封面 | project 部分(10个字段) + document 部分(7个字段) |
| `project.json` | 需求详细配置 | 模板信息 + 章节结构 + 占位符定义 |
#### 2.2.3 Interaction Layer(问答层)
问答层负责与用户的交互:
- 使用 AskUserQuestion 工具收集信息
- 根据字段类型决定填写方式:
  - `original`: 直接使用用户输入
  - `text_generation`: 根据描述生成文本
  - `content_generation`: 根据提示生成结构化内容
#### 2.2.4 Processing Layer(处理层)
处理层是核心业务逻辑所在,包含以下模块:
| 模块 | 职责 |
|------|------|
| `main.py` | 主处理流程编排和命令行入口 |
| `doc_fill.py` | Word 文档填充 |
| `find_and_rep.py` | 文本查找替换(保持格式) |
| `dynamic_process.py` | 动态章节处理 |
#### 2.2.5 Template Layer(模板层)
模板层定义数据结构模板:
- `config_template.json`: 配置模板
- `project_template.json`: 项目模板
#### 2.2.6 Document Layer(文档层)
文档层包含 Word 模板文件:
- `[08]软件需求规格说明-438C.docx`: GJB 438C 标准 SRS 模板

### 3.2 文档处理组件
#### 3.2.1 处理流程
```
┌─────────────────┐
│   加载模板文档   │
│   Document()    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  封面信息替换    │
│  config.json    │
│  document 部分   │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  正文内容替换    │
│  project.json   │
│  structure 部分  │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  动态章节处理    │
│  subsections    │
└────────┬────────┘
         │
         ▼
┌─────────────────┐
│  保存输出文档    │
└─────────────────┘
```
#### 3.2.2 替换策略
系统采用多级替换策略:
1. **封面替换**: 根据 `config.json` 的 `document` 部分替换封面占位文本
2. **静态替换**: 直接匹配 `replacement_pattern` 进行文本替换
3. **动态替换**: 根据 `generation_prompt` 生成内容后替换,并处理 `subsections`
### 3.3 问答组件
#### 3.3.1 AskUserQuestion 工具
系统使用 Claude Code 的 AskUserQuestion 工具进行交互式信息收集:
```
用户输入
    │
    ▼
AskUserQuestion (交互问答)
    │
    ├─► 根据字段 type 决定处理方式
    │         │
    │         ├─ original ──► 直接填入 content
    │         │
    │         ├─ text_generation ──► 结合 description 生成
    │         │
    │         └─ content_generation ──► 结合 generation_prompt 生成
    │
    ▼
config.json / project.json
```

### 5.2 数据接口

```

```
## 6. 错误处理
### 6.1 错误分类
| 错误级别 | 来源 | 处理方式 |
|----------|------|----------|
| Critical | 文件缺失、格式错误 | 终止执行,提示用户 |
| Error | 验证失败、数据无效 | 回退修复,重新执行 |
| Warning | 占位符残留、文件过大 | 记录日志,继续执行 |
### 6.2 错误传播
```
┌─────────────┐
│  脚本执行   │

       │

       │
```

## 7. 扩展设计
### 7.1 模板扩展
支持添加新的文档模板:
1. 创建新的 Word 模板文件
2. 定义对应的 JSON 模板
3. 更新 `template_info` 配置
### 7.2 字段扩展
支持在模板中添加新的占位符:
1. 在 Word 模板中添加占位文本
2. 在 JSON 模板中定义字段结构
3. 设置 `replacement_pattern` 和 `content`
### 7.3 章节扩展
支持添加新的章节:
1. 在 `structure` 中添加新章节编号
2. 定义 `placeholders` 列表
3. 支持动态章节(使用 `X` 作为可变编号)
## 8. 性能考虑
### 8.1 优化策略
- **批量替换**: 合并同类替换操作
- **延迟加载**: 按需读取文档内容
- **格式缓存**: 缓存段落格式信息
### 8.2 资源限制
| 资源 | 限制 |
|------|------|
| 文档大小 | 建议 < 50MB |
| JSON 深度 | 建议 < 5 层 |
| 占位符数量 | 建议 < 100 个 |
| 子章节数量 | 建议 < 50 个 |
## 9. 安全考虑
### 9.1 数据安全
- 敏感信息不应硬编码在模板中
- 生成的文档应妥善保管
- 配置文件可能包含项目敏感信息
### 9.2 输入验证
- 所有用户输入需经过验证
- 文件路径需进行合法性检查
- JSON 解析需捕获异常
